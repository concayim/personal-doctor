# Personal Doctor Agent — 需求文档

> 版本：2026-06-03

## 1. 项目概述

一个个人医生聊天助手原型，支持管理多病人档案、录入病历/药方记录、按病人维度独立聊天，并把病历上下文自动注入医生助手的提示词中。

- 技术栈：Go 后端 + SQLite + Eino (模型编排) + Vue 3 前端
- 目标用户：个人健康管理场景，辅助整理病情、提醒注意事项、解释医学概念

## 2. 功能需求

### 2.1 病人档案管理

| 需求 | 说明 | 优先级 |
|------|------|--------|
| 创建病人 | 录入姓名、性别、生日、联系方式、过敏史 | P0 |
| 查看病人列表 | 按更新时间倒序，附带最近一次病历时间 | P0 |
| 选择/切换病人 | 点击病人卡片即切换，同时更新病历和聊天视图 | P0 |
| 刷新病人列表 | 手动刷新按钮，加载最新数据 | P1 |

### 2.2 病历和药方记录

| 需求 | 说明 | 优先级 |
|------|------|--------|
| 创建病历/药方 | 录入 kind (condition/prescription/exam/note)、标题、内容，自动记录时间 | P0 |
| 查看病历列表 | 按记录时间倒序展示当前病人所有病历 | P0 |
| 病历类型分类 | 病情、药方、检查、备注四种类型，前端以标签区分 | P1 |

### 2.3 医生助手聊天

| 需求 | 说明 | 优先级 |
|------|------|--------|
| 发送消息 | 向 AI 助手发送文本消息，支持 Enter 快速发送 | P0 |
| 接收回复 | AI 助手返回中文回复，内容参考病历上下文 | P0 |
| 聊天历史 | 按创建时间正序展示当前病人完整聊天记录 | P0 |
| 加载提示 | 发送时按钮置灰禁用，防止重复提交 | P1 |
| 错误提示 | 操作失败时底部 toast 通知 | P1 |

### 2.4 病历上下文注入

| 需求 | 说明 | 优先级 |
|------|------|--------|
| 病人信息注入 | 系统提示中包含当前病人姓名、性别、生日、过敏史、备注 | P0 |
| 病历记录注入 | 最近的 20 条病历/药方记录自动附在提示词中 | P0 |
| 安全边界提示 | 系统提示包含不替代诊断、急症就医等安全说明 | P0 |

## 3. 非功能需求

### 3.1 数据存储

- 使用 SQLite 作为本地数据库
- 外键约束必须启用 (`PRAGMA foreign_keys = ON`)
- `ON DELETE CASCADE` 确保删除病人时级联清理病历和聊天记录
- 病人、病历、聊天消息三个表独立存储

### 3.2 模型集成

- Mock 模式：未配置 API 时本地生成占位回复，供前端体验流程
- OpenAI 模式：通过 Eino 调用 OpenAI-compatible API，支持自定义 base URL
- Base URL 自动规范化，兼容根地址和 `/v1` 路径

### 3.3 安全性

- API key、model 等敏感配置只放在后端 `backend/.env`
- 前端不接触任何模型凭据
- `.gitignore` 忽略 `.env` 和 `data/` 目录

### 3.4 前端体验

- 三栏布局：左侧病人列表 | 中栏病历和药方 | 右侧问诊聊天
- 响应式设计，1080px / 640px 两个断点适配平板和手机
- 所有列表接口响应兼容空值，前端统一规范化为数组，避免页面崩溃
- 发送失败时输入框内容保留（不提前清空）

### 3.5 接口规范

- RESTful JSON API
- 错误响应使用统一 `{"error": "描述"}` 格式
- CORS 跨域支持，允许开发环境前后端分离部署

## 4. 数据模型

### 4.1 Patient（病人）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 主键 |
| name | string | 姓名 |
| gender | string | 性别 |
| birthday | string | 生日 |
| phone | string | 联系方式 |
| allergies | string | 过敏史 |
| notes | string | 备注 |
| createdAt | datetime | 创建时间 |
| updatedAt | datetime | 更新时间 |
| lastRecordAt | datetime/null | 最近一次病历时间（查询时计算） |

### 4.2 MedicalRecord（病历/药方）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 主键 |
| patientId | string | 关联病人 ID（外键） |
| kind | string | 类型：condition/prescription/exam/note |
| title | string | 标题 |
| content | string | 内容 |
| recordedAt | datetime | 记录时间 |
| createdAt | datetime | 创建时间 |

### 4.3 ChatMessage（聊天消息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 主键 |
| patientId | string | 关联病人 ID（外键） |
| role | string | user/assistant |
| content | string | 消息内容 |
| createdAt | datetime | 创建时间 |
